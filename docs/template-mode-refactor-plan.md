# 模板模式改造与 Bug 修复计划

## 目标

把现有应用整理为三块独立体验：

1. 生图模式
2. 模板模式
3. 收藏夹

同时修复当前几个关键问题：

- 创建模板时，不应再在普通生图画廊里多出一张模板卡
- 模板多选后，要能真正批量复用
- 选择模板后，需要明确提示导入额外参考图 / 材质图
- 卡片提示词显示要改成单行省略号
- 模板标签颜色功能要恢复可用

## 现状判断

项目里已经有一些底座，但还没串成完整流程：

- `TaskRecord.kind = 'template'` 已存在
- `templatePickerTaskIds / openTemplatePicker / closeTemplatePicker` 已存在
- 收藏夹、ZIP 导入导出、任务体系都已存在
- 普通任务列表已经尝试过滤模板，但展示和保存链路仍然串线
- `Agent` 仍然是主入口之一，需要从主导航中移除

## 最终产品形态

### 1. 生图模式

原来的“画廊”改名为“生图模式”。

职责：

- 普通图片生成
- 任务列表浏览
- 单任务复用 / 删除 / 收藏

要求：

- 不显示模板卡
- 不和模板模式混在一起

### 2. 模板模式

新增独立入口“模板模式”。

职责：

- 浏览模板卡
- 多选模板
- 批量复用模板
- 从当前输入保存模板

要求：

- 模板卡只在模板模式中出现
- 支持单槽模板兼容旧数据
- 支持多参考图 / 多替换位

### 3. 收藏夹

收藏夹保持独立入口。

要求：

- 继续支持收藏任务
- 继续支持默认收藏夹
- 不和模板模式合并视图

## 需要做的改造

### 导航与模式

- 删除 `Agent` 作为主导航入口
- 将原 `gallery` 改名为 `生图模式`
- 新增 `模板模式`
- 收藏夹保留独立入口

### 模板保存

从当前输入区保存模板时，需要支持：

- 提示词
- 1 张或多张参考图
- 指定哪些图片位是可替换位

默认规则：

- 仍沿用旧习惯，最后一张图作为替换位
- 兼容旧模板数据

### 模板批量复用

流程应为：

1. 在模板模式中多选模板
2. 点击批量复用
3. 弹出提示，要求导入额外参考图 / 材质图
4. 导入后开始批量生成

生成规则：

- 按模板保存时的提示词、参考图、替换位规则逐个复用
- 批量创建普通生图任务

### 展示与筛选

- 普通生图列表不显示模板卡
- 收藏夹视图只展示真实收藏内容
- 模板模式中才显示模板卡和模板批量操作

### 卡片 UI

- 提示词超出一行时，直接显示单行省略号
- 不要中间截断
- 模板卡和普通任务卡视觉上明确区分

### 标签颜色

- 恢复模板 / 任务卡的自定义标签颜色
- 确保颜色真正参与渲染
- 刷新与导入导出后仍能保持

## 建议实现顺序

### 第一阶段

- 新增模板模式入口
- 移除 Agent 主导航入口
- 将画廊命名改成生图模式
- 保留收藏夹独立入口

### 第二阶段

- 修复模板保存不污染普通画廊
- 修复模板卡误混入收藏夹 / 普通任务列表
- 让模板卡只在模板模式展示

### 第三阶段

- 接通模板多选后的批量复用流程
- 增加“导入额外参考图 / 材质图”的提示
- 让批量生成真正落到任务创建链路

### 第四阶段

- 修复提示词单行省略号
- 修复标签颜色持久化与渲染
- 做导入导出回归

## 验收标准

- 创建模板后，普通生图画廊不再多出模板卡
- 模板模式里可以看到模板卡
- 收藏夹和模板模式互不串台
- 模板多选后可以进入批量复用流程
- 会提示导入额外参考图 / 材质图
- 导入后可以批量生成
- 提示词卡片显示为单行省略号
- 标签颜色可正常生效并持久化
- 旧 ZIP 导入导出仍可正常工作

## 备注

- 继续复用现有任务体系和 ZIP 结构，不新增独立模板包
- Agent 数据可以保留兼容，但不作为主入口
- 先保证功能可用，再做交互细化
